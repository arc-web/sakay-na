The problem

What's broken without this

😤
Jeepney routes require local knowledge
New commuters waste time asking strangers which jeepney numbers connect their origin to destination, especially for multi-transfer routes.
💸
Vehicle owners guess fuel timing
Jeepney and tricycle operators make refueling decisions without data, often filling up at peaks or running dangerously low.
🔁
API rate limits kill navigation apps
External route APIs throttle requests during peak commute hours, leaving users stranded without directions when they need them most.
The difference

Why it matters

✗ Without
Manual work
Slow iteration
Repeated mistakes
✓ With this
Runs entirely offline—no API dependency means zero latency during rush hour
Understands Cebu-specific location aliases ('Mabolo' = 'SM City') that generic maps don't recognize
Detects multi-transfer routes by connecting through known hub intersections, not just point-to-point directions
Under the hood

How it works

📥
User enters start and end location
Commuter types origin (e.g., 'Mabolo') and destination (e.g., 'Ayala') into the search box; system recognizes colloquial aliases.
⚙️
Local matching engine evaluates routes
Algorithm scans ingested jeepney route database for direct matches, then bridges multi-leg journeys through major hubs like Colon and IT Park without external API calls.
📤
Display jeepney numbers and transfer points
User receives immediate list of jeepney numbers to ride in sequence, with landmark names and estimated transfer locations.
